1. System Architecture

1.1 High-Level Architecture
The system follows a decoupled architecture: a Next.js 14 frontend (App Router) communicates with an Express.js REST API backend, with MongoDB Atlas as the primary datastore. JWT-based authentication spans both layers.

1.3 Data Flow (End-to-End)
Request flow for a student submitting a pass application:
Student fills form in Next.js UI and uploads documents to Cloudinary
Frontend calls POST /api/applications with JWT in Authorization header
Express auth middleware verifies JWT, extracts userId and role
RBAC middleware checks role === 'student'
Controller validates payload (Zod), creates PassApplication document
Admin receives notification; approves via PATCH /api/applications/:id
PassService generates Pass document and HMAC-signed QR payload
Student fetches pass; QR rendered in browser via qrcode library
Conductor scans QR via camera; payload POSTed to /api/verify
VerifyService decodes, validates HMAC, checks pass status and bus assignment, returns result
VerificationLog document created for audit trail

3. Authentication & RBAC

3.1 JWT Strategy
Two-token strategy: short-lived access token + long-lived refresh token stored in HTTP-only cookie.

// services/TokenService.js
const generateTokens = (userId, role) => ({
  accessToken:  jwt.sign({ sub: userId, role }, process.env.ACCESS_SECRET,  { expiresIn: '15m' }),
  refreshToken: jwt.sign({ sub: userId },      process.env.REFRESH_SECRET, { expiresIn: '7d'  }),
});
3.2 Auth Middleware
// middleware/auth.js
export const authenticate = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'No token' });
  try {
    req.user = jwt.verify(token, process.env.ACCESS_SECRET);
    next();
  } catch { res.status(401).json({ error: 'Invalid token' }); }
};

// middleware/rbac.js
export const authorize = (...roles) => (req, res, next) => {
  if (!roles.includes(req.user.role))
    return res.status(403).json({ error: 'Forbidden' });
  next();
};
3.3 Protected Routes (Next.js)
// middleware.ts  (Next.js root middleware)
export function middleware(request: NextRequest) {
  const token = request.cookies.get('accessToken')?.value;
  const { pathname } = request.nextUrl;

  const roleRoutes = {
    '/admin':     'super_admin',
    '/student':   'student',
    '/conductor': 'conductor',
  };

  const requiredRole = Object.entries(roleRoutes)
    .find(([path]) => pathname.startsWith(path))?.[1];

  if (!requiredRole) return NextResponse.next();
  if (!token) return NextResponse.redirect(new URL('/login', request.url));

  const payload = verifyToken(token);
  if (!payload || payload.role !== requiredRole)
    return NextResponse.redirect(new URL('/login', request.url));

  return NextResponse.next();
}

4.3 Pass Generation (QR Logic)
// services/PassService.js
const generate = async (applicationId) => {
  const app = await PassApplication.findById(applicationId)
    .populate('student assignedBus assignedRoute');

  const payload = {
    passId:    new mongoose.Types.ObjectId().toString(),
    studentId: app.student._id.toString(),
    busId:     app.assignedBus._id.toString(),
    exp:       addMonths(new Date(), 6).getTime(),
  };

  const hmac = crypto.createHmac('sha256', process.env.QR_SECRET)
    .update(JSON.stringify(payload))
    .digest('hex');

  const qrToken = Buffer.from(JSON.stringify({ ...payload, hmac })).toString('base64url');

  return Pass.create({
    student:       app.student._id,
    application:   applicationId,
    assignedBus:   app.assignedBus._id,
    assignedRoute: app.assignedRoute._id,
    qrToken,
    issuedAt:  new Date(),
    expiresAt: addMonths(new Date(), 6),
    autoRenew: app.autoRenew,
  });
};
4.4 QR Verification Flow (HMAC Validation)
Conductor opens scanner screen (/conductor/scan)
Browser requests camera; uses jsQR or html5-qrcode library
Decoded QR string POSTed to POST /api/verify
VerifyService decodes base64url, parses JSON
Recomputes HMAC on { passId, studentId, busId, exp } — compares with hmac field
If HMAC mismatch: return { result: 'invalid', reason: 'tampered' }
If exp < Date.now(): return { result: 'invalid', reason: 'expired' }
Fetch Pass from DB by passId — check status not 'suspended'/'revoked'
Check pass.assignedBus === conductor's assignedBus — prevents bus hopping
If all checks pass: return { result: 'valid', student: { name, universityId } }
Create VerificationLog entry regardless of result
Update pass.lastScannedAt on valid scan
4.5 Auto-Renewal System
// jobs/autoRenew.js  (runs daily via node-cron)
cron.schedule('0 2 * * *', async () => {
  const soon = addDays(new Date(), 7); // expires in 7 days
  const passes = await Pass.find({
    autoRenew: true,
    status: 'active',
    expiresAt: { $lte: soon }
  }).populate('student application');

  for (const pass of passes) {
    // Create new application pre-approved
    const newApp = await PassApplication.create({
      student:       pass.student._id,
      status:        'approved',
      documents:     pass.application.documents,
      assignedBus:   pass.assignedBus,
      assignedRoute: pass.assignedRoute,
      autoRenew:     true,
    });
    await PassService.generate(newApp._id);
    await Pass.findByIdAndUpdate(pass._id, { status: 'expired' });
  }
});

8. QR System Design

8.1 Token Structure
The QR token is a base64url-encoded JSON object containing:
{
  passId:    string,   // MongoDB ObjectId of Pass document
  studentId: string,   // MongoDB ObjectId of User document
  busId:     string,   // MongoDB ObjectId of assigned Bus
  exp:       number,   // Unix timestamp (ms) — pass expiry
  hmac:      string,   // HMAC-SHA256 hex of above 4 fields
}
8.2 HMAC Signing
const payload = { passId, studentId, busId, exp };
const hmac = crypto
  .createHmac('sha256', process.env.QR_SECRET)
  .update(JSON.stringify(payload))   // deterministic field order
  .digest('hex');

const qrToken = Buffer
  .from(JSON.stringify({ ...payload, hmac }))
  .toString('base64url');             // URL-safe, no padding
8.3 Anti-Tampering Verification
const verify = (qrToken) => {
  const raw = JSON.parse(Buffer.from(qrToken, 'base64url').toString());
  const { hmac, ...data } = raw;

  // Recompute HMAC — same key, same field order
  const expected = crypto
    .createHmac('sha256', process.env.QR_SECRET)
    .update(JSON.stringify(data))
    .digest('hex');

  // Timing-safe comparison prevents timing attacks
  if (!crypto.timingSafeEqual(
    Buffer.from(hmac), Buffer.from(expected)
  )) return { valid: false, reason: 'tampered' };

  if (data.exp < Date.now()) return { valid: false, reason: 'expired' };

  return { valid: true, data };
};
8.4 Optional: Rotating QR
For higher security, QR tokens can rotate every 60 seconds. Implementation approach:
Pass document stores a qrSeed (random bytes) alongside the HMAC key
Frontend fetches a fresh QR token via GET /api/passes/my/qr every 55 seconds
Token includes iat (issued-at, minute-resolution) — server rejects tokens older than 2 minutes
Conductor must have network connection to validate rotating tokens
Note: Static QR is recommended for offline-capable scenarios. Rotating QR is opt-in.

export const validate = (schema) => (req, res, next) => {
  const result = schema.safeParse(req.body);
  if (!result.success) return res.status(400).json({ errors: result.error.flatten() });
  req.body = result.data;
  next();
};

  const message = err.message || 'Internal Server Error';
  logger.error({ status, message, stack: err.stack, url: req.url });
  res.status(status).json({ error: message });
};
